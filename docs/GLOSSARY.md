# Glossary

*Alphabetical. Each entry is short and links to the doc that owns the detail. New
recurring term in the repo ‚Üí new entry here, same change (registry trigger).*

**Advanced (fidelity tier)** ‚Äî The expensive, opt-in re-conversion of a single
source (`fux ingest --advanced <file>`): Docling layout extraction for office/PDF,
Tesseract OCR for images. Flips the file's [fidelity](#fidelity) from `inferred` to
`advanced`; survives re-ingests until the source changes. See
[ingest-strategy](compare/ingest-strategy.compare.md).

**Anton (AlphaForge)** ‚Äî Arpit's trading app; Fux's pilot project ("instance
zero") and the litmus for priorities: *is it relevant to Anton?* Everything ships
first-hand there before any external claim. See [CLAUDE.md](../CLAUDE.md).

**ADR** ‚Äî Architecture Decision Record. One per completed feature, in
[`adr/`](adr/): decision, context, alternatives, consequences, references
(required). See the lifecycle in [CLAUDE.md](../CLAUDE.md).

**AGENTS.md** ‚Äî The Linux-Foundation-stewarded open standard for project
instructions that most coding agents read natively. Fux's canonical agent contract;
tool-native files (CLAUDE.md section, copilot-instructions, Kiro steering) are thin
pointers to it. Generated by `fux setup --agents`. See
[agent-integration](compare/agent-integration.compare.md).

**BM25F** ‚Äî The lexical ranking function of the engine: BM25 with *fielded* term
frequency ‚Äî heading (weight 3.0), file path (2.0), body (1.0) summed first, then
saturated once (k1=1.2, b=0.75). Not per-field BM25 glued together. See
[query-engine](compare/query-engine.compare.md).

**Bundle (OKF)** ‚Äî A directory tree of Markdown files with YAML frontmatter, the
OKF unit of distribution. Both `docs/` and the ingest cache are bundles. See
[OKF](#okf-open-knowledge-format).

**Cache (`.fux/cache/`)** ‚Äî The converted, chunk-ready Markdown mirror of your
sources ‚Äî an OKF bundle with provenance frontmatter, designed to be **committed to
git** (it's the [corpus](#corpus), not scratch). Deterministic output is a hard
requirement. See [ingest-strategy](compare/ingest-strategy.compare.md).

**CDP (Chrome DevTools Protocol)** ‚Äî JSON-RPC over WebSocket exposed by Chrome
(`--remote-debugging-port`); how `render = "cdp"` sources capture the *rendered*
DOM of JS-heavy pages. Fux drives the user's own headless Chrome via a hand-rolled
[RFC 6455](#rfc-6455) client ‚Äî no bundled browser. See
[ingest-strategy](compare/ingest-strategy.compare.md).

**Chunk** ‚Äî The retrieval unit: a heading-bounded slice of a cached document
(256‚Äì512 token target, code fences/tables atomic) carrying its heading path and
source `file:line` span. See [query-engine](compare/query-engine.compare.md).

**Compare doc** ‚Äî A decision record written *before* building whenever a fork has
multiple viable options: debate, matrix, grounded references, proposed verdict
Arpit accepts or overrides, and a reopen-trigger. Lives in [`compare/`](compare/).

**Corpus** ‚Äî The long-term, git-versioned knowledge asset Fux builds and queries:
cache + manifest (committed) with the index derived from it. "A corpus, not a
disposable index" ‚Äî see [fux-plan ¬ß6b](fux-plan.md).

**Determinism** ‚Äî Same sources ‚Üí byte-identical cache, manifest, index ‚Äî and the
same question ‚Üí the same answer. No wall-clock output, no model in the maintenance
path, sorted walks, stable serialization. The property behind trust, goldens, and
the audit story. See [CLAUDE.md](../CLAUDE.md).

**Distillation** ‚Äî The dev-time-only pipeline (`tools/distill/`) that re-packs a
teacher embedding model into Fux's bundled artifact: vocab-trimmed, int8-quantized,
7.93 MB, sha-pinned. Heavy deps allowed there; never at runtime. See
[packaged-model](compare/packaged-model.compare.md).

**Docling** ‚Äî IBM Research's layout-aware document converter (tables, multi-column,
built-in OCR stage). Fux's *advanced-tier* converter for office/PDF; opt-in extra,
never a runtime dep. See [advanced](#advanced-fidelity-tier).

**DOGFOOD.md** ‚Äî The repo-root 10-minute quickstart for pointing Fux at a real
project (Anton first). Dogfooding is the gate for external claims.

**Drift** ‚Äî A source file changing (or appearing/disappearing) after its cache
entry was written; detected by sha comparison via `fux ingest --check` (advisory;
`--strict` exits 2). The "don't let knowledge silently go stale" mechanism.

**Eval harness** ‚Äî Committed question‚Üíexpected-passage pairs + metrics
(hit@1/hit@5/[MRR](#mrr)) run against the engine (`tests_e2e/eval/`). Both the
ship-gate for retrieval changes and the recorded reopen-instrument for closed
decisions (reranker, weights). Lexical baseline: hit@1 0.762 ¬∑ hit@5 0.952 ¬∑ MRR
0.833. See [handoff 0003 ‚Üí archive](archive/).

**Extractive answer** ‚Äî `fux answer`'s output: verbatim source sentences selected
(passage score √ó question overlap √ó [TextRank](#textrank) centrality), ordered,
each cited `[n] path:line`. Never generative ‚Äî a ‚â§10 MB model can't write faithful
prose, so Fux selects instead of synthesizing. See
[query-output](compare/query-output.compare.md).

**Fail-open** ‚Äî The hook contract: a hook error must never break the agent's
session ‚Äî but never silently either (traces under `FUX_DEBUG=1`). See
[agent-integration](compare/agent-integration.compare.md).

**Fidelity** ‚Äî Per-file conversion-quality marker in frontmatter + manifest:
`inferred` (fast default pass) or `advanced` (Docling/OCR upgrade). What lets an
agent *know* it's reading a cheap extraction and request better. See
[ingest-strategy](compare/ingest-strategy.compare.md).

**Frontmatter** ‚Äî The YAML metadata block atop a Markdown file. Parsed and written
by Fux's hand-rolled, stdlib-only subset parser ‚Äî deliberately not PyYAML; that's
the zero-dependency promise. OKF requires a `type` key on every knowledge doc.
(Repo convention: ALL-CAPS docs ‚Äî GLOSSARY.md, DOC-REGISTRY.md ‚Äî are entry-point
files exempt from frontmatter, like CLAUDE.md/README.md.)

**Goldens** ‚Äî Committed expected outputs (`tests_e2e/goldens/`, normalized JSON)
that the real CLI's output must match byte-for-byte. Updated deliberately on
intentional behaviour change, never regenerated blindly. Derive from
[cli-examples.md](cli-examples.md).

**Handoff** ‚Äî A self-contained build spec (context, definition-of-done, scope
in/out, constraints, edge cases, tests, open questions) paired with a paste-ready
Claude Code **prompt**. Live in [`handoff/`](handoff/), archived on completion.

**Inferred (fidelity tier)** ‚Äî The fast, `$0` default conversion pass: native
md/txt/code, stdlib-flattened JSON, fenced YAML, image metadata stubs,
MarkItDown-class office/PDF when the extra is present. See [fidelity](#fidelity).

**Index (`.fux/index/`)** ‚Äî The derived retrieval structures: BM25F postings +
chunk vectors (`vectors.bin`). Regenerable from the cache; gitignore-able. Not the
[corpus](#corpus).

**Manifest (`.fux/manifest.jsonl`)** ‚Äî One canonical JSON line per ingested file:
source path, sha256, fidelity, converter, cache path. The index's invalidation key
and `--check`'s ground truth.

**MarkItDown** ‚Äî Microsoft's fast, many-format document‚ÜíMarkdown converter; Fux's
*inferred-tier* office/PDF converter via the `[ingest]` extra.

**Model (bundled)** ‚Äî The 7.93 MB static-embedding model shipped *inside* the
wheel: token‚Üívector table distilled from potion-base-8M, int8, inferred in pure
stdlib (tokenize ‚Üí look up ‚Üí mean-pool; int32 accumulation). No download, no
service, no numpy. See [packaged-model](compare/packaged-model.compare.md).

**MRR** ‚Äî Mean Reciprocal Rank: average of 1/rank of the first correct result
across eval questions. One of the eval harness's gate metrics.

**OKF (Open Knowledge Format)** ‚Äî Google's open spec (v0.1, June 2026) for
knowledge as Markdown+frontmatter bundles: required `type`, `index.md` progressive
disclosure, `log.md` convention, citations, permissive consumption. Fux's docs and
cache conform. See [CLAUDE.md](../CLAUDE.md) ¬ßOKF.

**Passage** ‚Äî What `fux ask` returns: a ranked chunk rendered with `path:line`,
score, and heading context.

**Proposal** ‚Äî A parked idea kept with compare-doc rigor (`proposals/`,
`status: proposed`); graduates to a compare doc or plan entry when picked up.

**Provenance** ‚Äî The trail from any answer back to its origin: citation ‚Üí
cache frontmatter (source, sha256, converter, fidelity, url/parent/depth for web)
‚Üí git history. What makes answers auditable.

**RFC 6455** ‚Äî The WebSocket protocol. Fux implements a minimal client on stdlib
`socket`/`hashlib`/`base64` to speak [CDP](#cdp-chrome-devtools-protocol);
`websocket-client` exists only as a flagged fallback extra.

**RRF (Reciprocal Rank Fusion)** ‚Äî How lexical and semantic rankings combine in
the hybrid engine: each retriever contributes `1/(k+rank)` (k=60), summed ‚Äî ranks,
not scores, so no calibration problem. *This is Fux's reranking layer*; a
cross-encoder reranker was researched and rejected (~80 MB-class, 8√ó over budget;
reopen only on eval evidence). `--lexical-only` bypasses it byte-identically. See
[query-engine](compare/query-engine.compare.md).

**SKILL.md (Agent Skills)** ‚Äî The Anthropic-originated open standard for packaged
agent workflows, read by 32+ tools (Copilot, Kiro, Codex, Cursor‚Ä¶). Fux ships two
skills ‚Äî `fux-query` and `fux-ingest` ‚Äî written once, placed by
`fux setup --skills`. See [agent-integration](compare/agent-integration.compare.md).

**Static embeddings** ‚Äî Embeddings from a fixed token‚Üívector lookup table
(Model2Vec/Potion family): no neural forward pass at inference, hence stdlib-fast
on CPU. The only model class that fits Fux's ‚â§10 MB / zero-dep budget.

**Tesseract** ‚Äî The leading open-source OCR engine (offline, 100+ languages),
called as a subprocess in the advanced tier to turn images/scans into searchable
text.

**TextRank** ‚Äî PageRank over a sentence-similarity graph: sentences central to
their passage score higher. One of the three factors selecting
[extractive-answer](#extractive-answer) sentences ‚Äî an algorithm, not a model.

**Worklog** ‚Äî [`worklog.md`](worklog.md): the per-exchange rolling session
handoff (OKF log.md style, newest first) so a new chat picks up cold. Distinct
from [implementation.md](implementation.md) (live build board) and
[model-handoff-interview.md](model-handoff-interview.md) (succession judgment).

**`$0` (the zero-dollar law)** ‚Äî Fux's founding constraint: no third-party runtime
dependencies, no API spend, no model in the maintenance path ‚Äî enforcement and
retrieval must be free, offline, and auditable forever. See
[CLAUDE.md](../CLAUDE.md).
