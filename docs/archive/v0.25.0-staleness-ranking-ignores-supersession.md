# Retrieval ignores supersession

## What was measured

acme-payments (fux 0.23.0), 12 planted stale-vs-current pairs — an old doc says X,
a newer authoritative one says not-X.

- Current (still-true) doc in top-5: **10/12** — recall is fine.
- Superseded doc **outranks** current: **9/12** — almost always stale at rank 1,
  current at rank 2. An agent taking top-1 gets the retired answer 9/12 of the time.
- **All three markers fail equally:** `superseded_by:` frontmatter, a dated inline
  "*Superseded …*" note, and no marker.

evidence: `docs/conformance/2026-07-22-acme-payments/evidence/staleness-inversions.json`
repro: `fux find "What is the card settlement window today?" --json --top 10`
→ `docs/adr/0006-…-t-plus-three.md` (stale) rank 1 · `…0018-…-t-plus-two.md`
(current) rank 2.

## Why it happens

- BM25F length normalization favors the **terse legacy doc** (fewer, denser query
  terms) over the fuller current one.
- The dense plane cannot break the tie — the two docs are topically near-identical.
- Fused margin is razor-thin (settlement: 0.04892 vs 0.04813, Δ 0.00079).
- **Nothing in the pipeline knows one doc supersedes the other** — the markers are
  plain text it ranks on, not a signal it acts on.

## Why it matters

This is the product's core promise — *surface the answer that is still true*. On a
realistic repo with ordinary superseded docs, Fux surfaces the **wrong** one at
rank-1 most of the time. The 21-pair fixture gate never tested it (it plants no
stale pairs); the synthetic tiers can't (same). Only the realistic corpus caught it.

## Sketch — deterministic, no model, fits the substrate

1. **Parse supersession at ingest.** `status: superseded` / `superseded_by:` in
   frontmatter → a `superseded` flag on the chunk/doc in state. (Dated-inline and
   no-marker cases are out of reach without NLP — accept that; frontmatter is the
   machine-readable contract to lean on, and this run is a reason to prefer it.)
2. **Down-rank superseded chunks** by a fixed fusion penalty, or
3. **Annotate output** (`superseded: true`) so `answer` and agents prefer the
   current doc without changing `find` ordering.
4. **`fux why`** surfaces the flag, making an inversion self-explaining.

## Status

`implemented` — as **Option 3 (annotate)**, not Option 2 (down-rank): see
[`../compare/supersession-handling.compare.md`](../compare/supersession-handling.compare.md)
(accepted 2026-07-23) and [ADR-0013](../adr/0013-supersession-awareness.md).
**Option 2 (fusion down-rank) remains explicitly not shipped** — still gated on
a second realistic corpus confirming the effect and a tunable-without-regression
penalty, per that compare doc's reopen-trigger. `find`/`ask` ranking is
unchanged by this implementation; `answer` prefers the current document when
both are in its candidate pool. Recovery on the measured 9/12 is **partial** —
see `docs/conformance/2026-07-23-supersession-recovery/` for the real number.
