# Handoff: Supersession Down-Rank + Margin Re-measure (phase 7) → v0.26.0

**Model: Opus.** M2 (config knob, default-off, applied to a deterministic set) is
Sonnet-grade. **M3 and M4 are not:** deciding whether a penalty interval exists
that recovers inversions without regressing hit@5 across four eval sets, and
re-reading whether the margin check separates once the staleness confound is
removed, are judgment calls no test settles. "No safe interval exists" is again a
valid, valuable outcome — a model that picks a plausible penalty and ships it
reproduces the exact blind-tuning this project forbids.

**One-liner:** v0.25.0 gave the engine a supersession *signal* but not the
*authority to act on it in `find`* — so it correctly annotates a document as
retired and still ranks it #1. Two corpora now justify testing the deferred
down-rank penalty; this phase builds it default-off, calibrates it, and uses it
to give the refuted margin check a clean re-measurement.

**Owner / executor:** Claude Code
**Status:** Ready to build — **blocked on M1: Arpit reopening Option B** in
`supersession-handling.compare.md` (its accepted verdict was "defer B").

**Stress-tested:** Phase 6 argued B down on three grounds; two corpora + the
v0.25.0 annotation proof update all three.
(1) *"B is a heuristic"* — weakened: the penalised set is **deterministic**,
exactly the docs whose author wrote `superseded_by:` (orbit surfaced 6/6 in
`--json`/`why`). Only the magnitude is tuned, and inversions flip on RRF edges as
thin as 0.0006 cosine, so a small bounded penalty likely suffices.
(2) *"one corpus"* — cleared: 9/12 (acme) and 8/12 (orbit, independent domain),
the reopen-trigger's ≥8/12 bar met exactly.
(3) *"blind regression risk"* — removed by construction: default 0.0 is
byte-identical to today, and the penalty ships enabled only if calibration proves
a safe interval across fixture+acme+orbit+synthetic.
**Residual risk (real, keep visible):** B *does* change `find` ordering — the
thing Option A deliberately avoided. So it stays behind Arpit's explicit reopen
(M1) and a hard safe-interval gate (M5), never auto-enabled. A pre-mortem also
surfaced: penalising a superseded doc that is itself the *best* answer to a
question *about the old decision* — mitigated by keeping it a rank penalty, not a
filter (the doc still appears, just lower), and by a test for that case.

## 1. Context & background

- v0.25.0 (phase 6) parses `superseded`/`superseded_by:` frontmatter, persists it,
  annotates `find`/`ask`/`why`, and makes `answer` prefer the successor.
- It deliberately **did not** reorder `find` (Option A). The acme + orbit runs
  measured the consequence: the current doc reaches top-5 but the *retired* doc
  ranks above it in 8–9 of 12 pairs, annotation notwithstanding.
- Mechanism (orbit `ANALYSIS.md`): in 6 of 8 inversions the current doc **wins
  BM25F outright** (up to 2×) and loses on a **thin dense edge** that RRF flips.
- **The coupling that makes this phase do double duty:** the orbit margin-check
  refutation was confounded — the smallest "answerable" margins (1e-05) came from
  a doc tying with *its own superseded twin*. Remove those ties (this penalty) and
  the runner-up margin deserves a clean re-measurement. **Finding 1's fix is the
  prerequisite to a fair Finding 2 verdict.**

## 3. Scope

**Out of scope (explicit):**

- **Any NLP/heuristic supersession detection** — frontmatter only, still.

  it if DoD 6 shows the *margin* now works, which is a new mechanism, not the

## 5. Technical approach (decided)

1. **Penalty, not filter.** Superseded docs stay retrievable; their RRF
   contribution is reduced. Preserves the "about the old decision" query.
2. **Default 0.0 = identity.** The knob is inert until proven; this is what makes
   the change safe to land before it is safe to enable.
3. **Deterministic set.** Only author-marked docs are touched.
4. **Calibrate, then gate.** The magnitude is a measurement across four eval sets,
   never a guess; ship-enabled needs a clean interval *and* human sign-off.
5. **Reuse the penalty to de-confound the margin** — the one experiment answers
   both open findings.

## 8. Edge cases & risks

- A superseded doc that is the genuine best answer to a question *about the
  retired decision* → penalty (not filter) keeps it reachable; add a test.
- A supersession **chain** A→B→C → only terminal C is un-penalised; A and B both
  down-ranked. Confirm against the phase-6 chain resolution.
- Penalty large enough to bury a doc that is *also* the best match for an
  unrelated query → caught by the four-way hit@5 regression gate.
- No safe interval exists → ship off, document, and the margin re-measure (DoD 6)
  still runs at the best-available magnitude to answer Finding 2.
