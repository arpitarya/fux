# Honest-decline is too permissive for well-formed queries

## What was measured

acme-payments (fux 0.23.0), 4 typed unanswerable questions (crypto policy, k8s
autoscaler, GraphQL SLA, SMS-OTP vendor) — nothing in the corpus answers them.

- **Gibberish control declines** (`answer "xyzzy plugh …"` → `answer=null`, 0
  sources). This is the *only* decline case the synthetic gate ever tested.
- **All 4 well-formed unanswerables fabricate** — a confident extractive answer
  with 1–5 sources from unrelated docs. E.g. *"What is Acme's policy on
  cryptocurrency…?"* → answer quoted from `0005-store-amounts-in-minor-units.md`,
  5 sources.

evidence: `docs/conformance/2026-07-22-acme-payments/evidence/unanswerable-fabrication.json`
· `…/gibberish-control-answer.json`
repro: `fux answer "What is the uptime SLA percentage for Acme's GraphQL API?" --json`

## Why it happens

- The decline path triggers on an (near-)empty candidate pool — i.e. (near-)zero
  lexical overlap. Gibberish shares no terms, so it declines.
- A fluent out-of-scope question shares enough **incidental** vocabulary ("policy",
  "API", "vendor", "configure") with *some* passage that the pool is non-empty and
  the extractive threshold admits it.
- The threshold is **relative** (best-of-pool), never **absolute** — so a weak best
  match still answers.

## Why it matters

For an agent-facing engine, a confident wrong answer is worse than a decline. This
is the "never fabricate" contract, and it holds only for gibberish today. Every
realistic out-of-scope question is a confabulation risk. The synthetic single-
gibberish gate structurally cannot see this.

## Sketch

1. **Absolute-confidence floor.** `answer` declines when the best extractive score
   (or top dense cosine) is below an absolute threshold, independent of pool size.
   ADR 0010 already records a 0.23–0.26 dense-noise band — a natural anchor.
2. **Margin check.** Decline when the top candidate does not clear the runner-up by
   a margin (off-topic pools are flat — the acme fabrications had 4–5 near-equal
   weak sources).
3. Keep the empty-pool early-return; add the floor **above** it.

## Status

`implemented` — the **mechanism**, not a working default. Built per
[`../compare/answer-decline-floor.compare.md`](../compare/answer-decline-floor.compare.md)
(accepted 2026-07-23, Option 1: absolute floor, no margin check) and
[ADR-0014](../adr/0014-answer-confidence-floor.md). Calibrated against all
five eval gates in `docs/conformance/2026-07-23-min-confidence-calibration/`:
**no single `min_confidence` value clears both the 4-unanswerable and
55-answerable gates** (interleaved score distributions, empty valid
interval) — shipped as `min_confidence = 0.0` (disabled) by default. **The
measured defect (0/4 decline) is not fixed in this release**; the follow-up
(calibrate on an absolute cross-query signal, e.g. dense cosine, instead of
the pool-relative sentence score) is filed in that conformance run's
ANALYSIS.md.
