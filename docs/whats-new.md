# What's new ‚Äî Fux release history

The full, detailed release notes. The [README](../README.md#whats-new) keeps only the
latest headline; everything lands here.

- **v0.17.3 ‚Äî direction-honest `savings` (fux-lab Cycle 2).** The harness's Cycle-2 pass (mutation/edge/determinism oracles) caught `fux savings` hard-coding the word `cheaper` and printing a negative `you save`: on a small corpus Fux's per-lookup overhead exceeds reading the file, so it claimed `1.9√ó cheaper` / `you save: -377 tok` for something **1.9√ó costlier**. Now a direction-aware comparison says `cheaper` only when Fux costs less (else `costlier`), and prints `you spend: N ‚Ä¶ extra` (positive) instead of a negative save. Regression test added; full suite 372 passed. `$0`/stdlib.
- **v0.17.2 ‚Äî error-contract consistency (fux-lab Cycle 1).** The harness's Cycle-1 pass (deepened with 20 behavioral oracles that assert each command's *documented effect*) showed the `why <unknown>` fix in 0.17.1 was **systemic**: `explain`/`path`/`query`/`seal`/`candidates`/`fetch-rules`/`ingest --connector` all printed `fux: <msg>` to *stdout* and returned 1, bypassing the `FuxError` boundary. All now raise `FuxError` ‚Üí terse `error: ‚Ä¶` on *stderr* (exit 1). `path` with valid-but-disconnected endpoints is now a legit empty result (exit 0). Regression tests parametrized over the affected commands; full suite 371 passed. `$0`/stdlib.
- **v0.17.1 ‚Äî harness-driven fixes: consistent error contract + seal-stable frontmatter.** A new exhaustive, offline, `$0` end-to-end test harness (the external **`fux-lab`** repo ‚Äî a polyglot fixture + a stdlib-only runner that drives *every* `fux/registry.py` command, diffs machine output against golden fixtures, springs worktree-isolated traps and negative guards, and proves `$0` with the network disabled) surfaced two real engine bugs, both fixed here: (1) **error contract** ‚Äî `fux why <unknown-id>` printed `fux: no rule '‚Ä¶'` to *stdout* and returned 1, bypassing the `FuxError` boundary; it now raises `FuxError` so `main()` renders the documented terse `error: ‚Ä¶` on *stderr*; (2) **seal stability** ‚Äî the frontmatter writer emitted a *string* `"true"` as bare `true`, which the reader coerced back to a bool, so a ratified rule's quoted-boolean example flipped its `content_seal` and raised a **false `tampered`** (a constitution-integrity false positive) ‚Äî the writer now quotes any string whose bare form would re-parse as a non-string, making `dump ‚Üí split` a fixed point. Regression tests added; the harness re-runs at **0 findings**. `$0`/stdlib, no behaviour change beyond the two fixes.
- **v0.17.0 ‚Äî Error-handling hardening: hooks fail-open, one `FuxError`, a documented exit-code contract.** fux now has a single error boundary ‚Äî the CLI `main()` ‚Äî so a missing `.fux/`, a bad rule id, or Ctrl-C renders as a terse `error: <msg>` instead of a raw traceback. Exit codes are a contract: `0` ok ¬∑ `1` error (`FuxError` for expected failures, or an unexpected exception) ¬∑ `2` blocking (strict `gate`/`stop`) ¬∑ `130` interrupted. Every hook entrypoint (`session_start`/`post_tool_use`/`stop`/`session_end_propose`/`user_prompt_recall`) is **fail-open** ‚Äî a hook error returns 0 so a broken `.fux/` can never break an agent session ‚Äî with the one exception of the deliberate strict `stop`‚Üí`2`, computed before the wrapper so it passes straight through. **Fail-open ‚âÝ fail-silent:** every swallowed hook exception (and the full CLI traceback) surfaces under `FUX_DEBUG=1`, the single new debug switch. The shell wrappers for the three non-blocking hooks guard `fux_run` with `|| true`; `stop.sh` deliberately does not, keeping its exit-2 passthrough. One thin `FuxError(Exception)` ([fux/errors.py](../fux/errors.py)) ‚Äî no hierarchy ‚Äî for expected user-facing failures; internals keep raising (boundary discipline). `$0`/stdlib, no behaviour change to derived views or checks. **Also bundled in this release:** the build-time **skill renderer** (`tools/skillgen` ‚Äî one fragment set ‚Üí the `fux` skill's claude/agents/copilot artifacts, with a `--check` drift-guard in CI), the **tier-scrutiny ADR** (`docs/decisions/0002`), and the **`parity` command decommission** (graph coverage subsumes it).
- **v0.16.0 ‚Äî Rule Proposer: agents author rules-with-why, humans ratify.** Closes the authoring loop ‚Äî the agent that wrote the code **proposes** the rules, you only ratify. Two surfaces, one persistent review file `.fux/CANDIDATES.md` (not the derived `DRIFT.md`; it never blocks). **Forward** (agent skill `propose-rules`): on an opt-in SessionEnd / per-PR pass the agent reviews the diff **plus its rationale** (commit/PR/session) and drafts candidates-with-why ‚Äî captured at decision-time while it's fresh ‚Äî then files them with `fux propose-rules --from drafts.json`. **Retro** (`fux propose-rules --retro`, `$0`): `fux mine` surfaces latent AST invariants and their *why* is recovered from the git commit subjects touching each site ‚Äî capped, deduped (vs existing rules **and** the queue), why-density-sorted. The **¬ß4 gate** keeps drafts honest: no recoverable why ‚áí no draft (a genuine invariant may draft `why: TODO`, sorted last; a pure code-restatement is dropped). `fux candidates [accept|reject] <id>` triages ‚Äî `accept` is the human ratification ‚Üí an active **standard** rule (constitutional candidates route via `/fux debate` ‚Üí `fux ratify`). **Hard lines:** every proposal is `status: draft`, `tier: standard` ‚Äî nothing auto-active, **nothing auto-constitutional**; the forward hook is opt-in (`propose_forward`). The agent spends the drafting tokens; the engine's gate/dedup/file step is `$0`/stdlib/deterministic and calls no model (guard test).
- **v0.15.0 ‚Äî PII content gate: a stray PAN/Aadhaar/account is caught in CI, not just locally.** `fux pii-scan` ports dante's BLOCK-tier regexes (PAN `[A-Z]{5}[0-9]{4}[A-Z]`, Aadhaar `\d{4} \d{4} \d{4}`, account/folio + a 6+ digit run) into a **stdlib** probe (no pip dependency on dante) and wires it as a step in the required **`fux gate`** CI job ‚Äî so a hard identifier in a non-plan `.py`/`.md` blocks the merge, closing the gap that a local hook leaves (`--no-verify` bypasses it). Plan/spec/handoff/decision docs are exempt by path (they discuss the identifiers as examples), and any single line can opt out with an inline `pii-allow` marker. `$0`, stdlib, deterministic ‚Äî no LLM, no network.
- **v0.14.0 ‚Äî connector ingestion: Jira/Confluence/GitHub ‚Üí the review queue.** `fux ingest --connector github|jira|confluence --query "<filter>"` lets the agent pull **structured, server-side-filtered** data through the existing MCP connectors / APIs ‚Äî **fux never builds a client or calls an API** ($0, client-free) ‚Äî then the same reduce ‚Üí draft ‚Üí review-queue ‚Üí govern pipeline runs. These are **low-trust** (`trust: candidate` ‚Äî a ticket/wiki/PR is not a spec): they land in the queue, never auto-active. The bounds are enforced by the engine: an **explicit query is mandatory** (an unbounded "everything" pull is refused), the count is capped, and `--since` does a delta. The skill carries the efficiency stack (server-side filter ‚Üí delta cursor ‚Üí structure-slice ‚Üí reduce/dedup, **GitHub first**) and the fallback ladder (MCP ‚Üí REST+PAT ‚Üí export/`git clone` ‚Üí CDP-JSON ‚Üí DOM scrape, probes last). `source_type` gains `jira`/`confluence`/`github`.
- **v0.13.0 ‚Äî decision capture: every debate becomes a tamper-evident, routed ADR.** When a `/fux debate` or decision-council concludes, `fux capture-decision <id> --route fux|anton|elgar` writes the verdict as an ADR sealed with `content_seal` (+ the debate transcript's `debate_hash`) ‚Äî `fux check` flags any later edit as `tampered`. Routing is **by content**: world/code/tool-design ‚Üí fux, app ‚Üí anton, and **money/personal figures ‚Üí elgar**, where the **firewall (ADR 0001)** holds: fux stores **only** `elgar://decision/<id>`, never the body, and a money route is refused without explicit `--yes`. A new always-blocking `firewall` finding catches any fux-side decision that isn't link-only ‚Äî a money figure can't leak into the public tree. Captured ADRs live in `.fux/decisions/`; the capture path is `$0`, stdlib, deterministic (the guard test covers it).
- **v0.12.0 ‚Äî fux explains fux: a self-knowledge bundle + `--self` scope.** `fux self-build` runs fux's own `$0`, AST-only graph extraction over the `fux/` package + its `.fux/rules` and ships the result as a pre-built bundle in the wheel. Point any graph query at it with `--self`: `fux explain --self check`, `fux query --self "drift detection seal"`, `fux path --self check.py seal.py`, `fux recall --self` ‚Äî answers come straight from fux's **real module relationships**, and it works in **any repo with no project `.fux/`**. The bundle regenerates **byte-identically** from source (a test fails CI if it drifts), so fux's self-knowledge can never lie about fux's code, and `meta.extractor` is pinned so it's identical with or without the `[ast]` extra. Still `$0`, stdlib, offline (the guard test covers `self-build` too).
- **v0.11.0 ‚Äî batch ingest + follow a page to the documents it links.** `/fux ingest` now takes **many** sources at once ‚Äî `fux ingest ./docs/*.pdf b.xlsx https://x ‚Ä¶` ‚Äî expands globs, dedups by `source_hash`, and is **partial-failure-tolerant** (one bad source is recorded `failed`; the batch continues). Output is a **draft review queue** (`.fux/ingest/queue.md`, shown by `fux ingest --queue`) ‚Äî nothing auto-activates. With `--follow-links`, when a URL is an HTML page the agent discovers the documents it links and ingests them under **hard bounds**: depth-1 only, same-origin (`--cross-origin` widens), an extension allow-list (never executables/scripts/archives), a `--max` cap (refuse-with-message), and list-and-confirm unless `--yes`. New source types ‚Äî `.docx`, `.json`, `.yaml`, and **Swagger/OpenAPI** (per-endpoint/param/auth/deprecation draft rules; `--recheck` flags contract drift). A `$0` **reduce-before-draft** pass trims the *extracted text* per type (Excel ‚Üí schema + sample rows, never the full grid) before the agent drafts, reporting tokens before‚Üíafter; `--full` bypasses it. The engine still imports no network/LLM/PDF/Excel/Word/OCR/vision library (a guard test proves it).
- **v0.10.0 ‚Äî ingest from files, not just URLs.** `/fux scrape` is now `/fux ingest <url|file>` ‚Äî same skill, generalized: the agent extracts a URL, PDF, Excel/CSV, TXT/Markdown, or image (its own pdf/xlsx/vision skills do the reading; the engine never imports one), classifies trust, and drafts `status: draft` rules carrying `source`/`source_type`/`fetched`/`source_hash`. An image- or scan-derived money or regulatory figure is flagged `verify-source` ‚Äî never auto-trusted. `--recheck` now re-verifies file sources too. `scrape` keeps working as a deprecated alias for one release. A guard test proves the engine still imports no PDF/Excel/OCR/vision library.
- **v0.9.0 ‚Äî web ‚Üí draft rules, and fux explains fux.** `/fux scrape <url>` lets the agent fetch a page (HTTP, escalating to **CDP** for client-rendered shells ‚Äî port configurable by flag/env/config, default `:9299`) and draft governed rules with `source`/`fetched`/`source_hash` provenance ‚Äî always `status: draft`, never auto-active, regulatory drafts flagged *verify-against-source*. `fux how "<question>"` dogfoods the engine: it runs the same `$0` BM25F recall over fux's **own** command registry and returns the exact command for the task (`fux how "which rules govern a file"` ‚Üí `fux refs <path>`). And one **command registry** now drives a grouped `fux --help`, `fux help <cmd>`, *and* the generated `docs/cli.md` ‚Äî so help and docs can't drift. The maintenance path stays model-free **and offline** (a guard test proves it).
